Generative Adversarial Networks // Redes generativas adversarias (GAN)
https://machinelearningmastery.com/what-are-generative-adversarial-networks-gans/

Las redes generativas adversarias son un enfoque al modelado generativo, utilizando métodos de deep learning (aprendizaje profundo) como las redes neuronales convolucionales.

El modelado generativo es una tarea de aprendizaje no supervisada en machine learning que implica descubrir y aprender las regularidades o patrones en los datos de entrada de forma que el modelo se pueda utilizar para generar  o crear nuevos ejemplos que plausiblemente podrían haberse extraído del conjunto de datos original.

Las GAN son una forma inteligente de entrenar un modelo generativo al enmarcar el problema como un problema de aprendizaje supervisado, con dos submodelos, el modelo generador que entrenamos para generar nuevos ejemplos y el modelo discriminador que intenta clasificar los ejemplos como reales (desde el dominio) o falsos (generados). Los dos modelos se entrenan juntos en un juego de suma cero, hasta que el modelo generador es engañado la mitad de las veces, lo que significa que el modelo generador está generando ejemplos plausibles.

Las GAN son un campo emocionante y que cambia rápidamente, cumplen la promesa de los modelos generativos en su capacidad para generar ejemplos realistas en una variedad de dominios problemáticos, sobre todo en tareas de traducción de imagen a imagen, como traducir fotos de verano a invierno o de día a la noche y en la generación de fotos fotorrealistas de objetos, escenas y personas, incluso humanos que no podemos decir que son falsos..

Aprendizaje supervisado vs no supervisado
	Un típico problema de machine learning involucra usar un modelo para realizar una predicción (ejemplo: el modelado predictivo).
	
	Esto requiere un conjunto de datos de entrenamiento que se usa para entrenar un modelo, compuesto por varios ejemplos, llamados muestras, cada uno con variables de entrada (X) y etiquetas de clase de salida (y). Un modelo se entrena mostrando ejemplo de entradas, prediciendo salidas y corrigiendo el modelo para que las salidas se parezcan más a las salidas esperadas.
	Esta corrección del modelo generalmente se conoce como una forma supervisada del aprendizaje o aprendizaje supervisado.

(Ejemplo de aprendizaje supervisado)

Ejemplos de problemas de aprendizaje supervisado incluyen clasificación y regresión y ejemplos de algoritmos de aprendizaje supervisado incluyen logistic regression, y random forest.

Hay otro paradigma de aprendizaje donde el modelo solo se le dan las variables de entradas (x) y el problema no tiene ninguna variable de salida (y).

Un modelo se construye extrayendo o resumiendo los patrones en los datos de entrada, No hay corrección del modelo, ya que el modelo no está prediciendo nada.


“El segundo tipo principal de machine learning es el enfoque de aprendizaje descriptivo o no supervisado. Aquí solo se nos dan entradas y el objetivo es encontrar “patrones interesantes” en los datos [...] Ese es un problema mucho menos definido, ya que no se nos dice qué tipos de patrones buscar, y no hay una métrica de error obvia para usar (a diferencia del aprendizaje supervisado, donde  podemos comparar nuestra predicción de y para un x dado al valor observado)”

La falta de corrección está generalmente referida  a una forma no supervisada de aprendizaje o aprendizaje no supervisado.
 
(Ejemplo de aprendizaje no supervisado)

Ejemplos de problemas de aprendizaje no supervisado incluyen clustering y modelado generativo, y ejemplos de algoritmos de aprendizaje no supervisado son los K-means y las redes generativas adversarias (GAN).

Modelado Discriminatorio vs Generativo
	En aprendizaje supervisado, nos podríamos ver interesados en desarrollar un modelo para predecir la etiqueta de clase dado un ejemplo de variables de entrada,

	Esta tarea del modelado predictivo se denomina clasificación.

	La clasificación también se conoce tradicionalmente como modelado discriminativo.
	

“... usamos los datos de entrenamiento para encontrar una función discriminante f(x) que asigna cada x directamente a una etiqueta de clase, combinando así las etapas de inferencia y decisión en un solo problema de aprendizaje”

	Esto se debe a que un modelo debe discriminar ejemplos de variables de entrada entre clases; debe elegir o tomar una decisión sobre a qué clase pertenece un ejemplo dado.

(Ejemplo de Modelado discriminativo)

	Alternativamente, los modelos no supervisados que resumen la distribución de las variables de entrada pueden usarse para crear o generar nuevos ejemplos en la distribución de entrada
Como tal, este tipo de modelos se conocen como modelos generativos.

(Ejemplo de modelado generativo)

Por ejemplo, una sola variable puede tener una distribución de datos conocida, como una distribución gaussiana o una forma de campana.
	Un modelo generativo puede ser capaz de resumir suficientemente esta distribución de datos y luego usarse para generar nuevas variables que encajen plausiblemente en la distribución de la variable de entrada.



“Los enfoques que modelan explícita o implícitamente la distribución de entradas y salidas se conocen como modelos generativos, porque al muestrear los es posible generar puntos de datos sintéticos en el espacio de entrada.”

Un modelo generativo realmente bueno puede generar nuevos ejemplos que no solo son plausibles, sino que no se pueden distinguir los ejemplos reales del dominio del problema.

Ejemplos de Modelos Generativos

Naive Bayes es un ejemplo de modelo generativo que es a menudo utilizado como un modelo discriminativo.

Por ejemplo, Naive Bayes, trabaja resumiendo la distribución de probabilidad de cada variable de entrada y la clase de salida. Cuando se hace una predicción, la probabilidad de cada resultado posible es calculado para cada variable, se combinan las probabilidades independientes y se predice el resultado más probable. Usada a la inversa, las distribuciones de probabilidad para cada variable se pueden muestrear para generar nuevos valores de características plausibles (independiente).
Otros ejemplos de modelos generativos incluyen Latent Dirichlet Allocation (LDA) y el  Gaussian Mixture Model (GMM).
Los métodos de deep learning pueden ser usados como método generativo. Dos ejemplos populares son Restricted Boltzmann Machine (RBM) y Deep Belief Network (DBN).
Dos ejemplos modernos de algoritmos  de modelado generativo de deep learning incluyen Variational Autoencoder (VAE) y las Generative Adversarial Network (GAN).

¿Qué son las redes generativas adversarias (GAN)?
	Las redes adversarias generativas o GAN, son un modelo generativo basado en deep learning.
	De forma más general, las GAN son una arquitectura de modelo para entrenar un modelo generativo, y es más común usar métodos de deep learning en esta arquitectura.
	La arquitectura de modelo GAN, involucra dos submodelos: un modelo generador para nuevos ejemplos y uno discriminador para clasificar  si los ejemplos generados son reales, del dominio, o falsos generados por el modelo generador.
	Generador : Modelo que es usado para generar nuevos ejemplos plausibles del domino del problema
	Discriminador: Modelo que es usado para clasificar ejemplos como reales (del dominio) o falsos (generados).
	
“Las redes generativas adversarias se basan en un escenario teórico de juegos en el que la red generadora debe competir contra un adversario. La red generadora produce muestras directamente. Su adversario, la red discriminadora, intenta distinguir entre las muestras extraídas de los datos de entrenamientos y las muestras extraídas del generador”

El modelo generador
	El modelo generador toma un vector aleatorio de longitud fija como entrada y genera una muestra en el dominio.
El vector se extrae aleatoriamente de una distribución Gaussiana, y el mismo se utiliza  para generar el proceso generativo. Después del entrenamiento, los puntos en este espacio vectorial multidimensional se corresponderán con puntos en el dominio del problema, formando una representación comprimida de la distribución de datos.	
Este espacio vectorial se denomina espacio latente o espacio vectorial compuesto por variables latentes. Las variables latentes o ocultas, son aquellas variables que son importantes  para un dominio pero que no son  directamente observables.
A menudo nos referimos a las variables latentes, o un espacio latente como una proyección o compresión de la distribución de datos. Es decir, un espacio latente proporciona una compresión o conceptos de alto nivel de los datos sin procesar como la distribución de los datos de entrada. En el caso de las GAN,  el modelo generador aplica significado a los puntos en un espacio latente elegido, de modo que los nuevos puntos extraídos de un espacio latente pueden proporcionarse al modelo generador como entrada y usarse para generar ejemplos de salida nuevos y diferentes.

“Los modelos de machine learning pueden aprender el espacio latente estadístico de imágenes, música e historias, y luego pueden tomar muestras de este espacio, creando nuevas obras de arte con características similares a las que el modelo ha visto en sus datos de entrenamiento.”	

Después del entrenamiento, el modelo generador se conserva y se usa para generar nuevas muestras.


El modelo discriminador
	El modelo discriminador toma un ejemplo del dominio como entrada (real o generada) y predice una etiqueta de clase binaria de real o falso (generado).
	El ejemplo real proviene del conjunto de datos de entrenamiento. Los ejemplos generados son generados por el modelo generador.
	El discriminador es un modelo de clasificación normal.
	Después del proceso de entrenamiento, el modelo discriminador es descartado ya que nos interesa el generador.
	A veces, el generador se puede reutilizar ya que ha aprendido de manera efectiva a extraer características de los ejemplos en el dominio del problema. Algunas o todas las capas de extracción de características pueden ser usadas para aplicaciones de aprendizaje de transferencia (transfer learning) usando los mismos o similares datos de entrada.

	
“Proponemos que una forma de construir buenas representaciones de imágenes es entrenando Redes Generativas Adversarias (GAN), y luego reutilizando las partes de las redes generadoras y discriminadoras como extractores de características para tareas supervisadas”


GAN como un juego de dos jugadores
	El modelado generativo es un problema de aprendizaje no supervisado, como fue planteado en anteriormente, aunque una propiedad inteligente de la arquitectura GAN es que el entrenamiento del modelo generativo se enmarca como un problema de aprendizaje supervisado.
Los dos modelos, el generador y el discriminador, son entrenados juntos. El modelo generador genera una serie de muestras y estas, junto con ejemplos reales del dominio son proporcionadas al discriminador y clasificados como reales o falsos. 
Luego el discriminador se actualiza para mejorar al discriminar muestras reales y falsas en la siguiente ronda y, lo que es más importante, el generador se actualiza basado en función de qué tan bien o no, las muestras generadas engañaron al discriminador.

	“Podemos pensar en el generador como si fuera un falsificador, tratando de hacer dinero falso, y al discriminador  como si fuera un policía, tratando de permitir dinero legítimo y atrapar dinero falso. Para tener éxito en el juego, el falsificador debe aprender como hacer dinero que sea indistinguible del dinero genuino, y la red generadora debe aprender a crear muestras extraídas de la misma distribución que los datos de entrenamiento. ”

	De esta manera, los dos modelos compiten entre sí, son adversarios en el  sentido de la teoría de juegos y juegan un juego de suma cero.


	“Debido a que el marco GAN puede ser naturalmente analizado con las herramientas de la teoría de juegos, llamamos a las GAN ‘adversarias’ ”.

	En este caso, suma cero significa que cuando el discriminador correctamente identifica muestras reales y falsas, es recompensado o no se necesita ningún cambio en los parámetros del modelo, mientras que el generador es penalizado con grandes actualizaciones en los parámetros del modelo.
	Alternativamente, cuando el generador engaña al discriminador es recompensado, o no se necesita ningún cambio  en los parámetros del modelo, pero el discriminador es penalizado y sus parámetros del modelo se actualizan.
	En un límite, el generador genera réplicas perfectas del dominio de entrada, y el discriminador no puede notar la diferencia y predice “Inseguro” (ejemplo: 50% para real y falso) en todos los casos. Este es solo un ejemplo de un caso idealizado, no es necesario llegar a este punto para lograr un modelo generador útil.
	
	


	








  









	“[el entrenamiento] lleva al discriminador a intentar aprender a clasificar correctamente las muestras como reales o falsas. Simultáneamente, el generador intenta engañar al clasificador haciéndole creer que sus muestras son reales. En la convergencia, las muestras del generador son indistinguibles de los datos reales y el discriminador genera ½ en todas partes. Entonces puede descartarse el discriminador.”

GAN y las CNN (Redes neuronales convolucionales)
	Las GAN suelen trabajar con datos de imágenes y utilizan redes neuronales convolucionales (CNN) como modelos generadores y discriminadores.
	La razón de esto puede deberse tanto a que la primera descripción de la técnica fue en el campo de la visión por computadora y utilizó CNN y datos de imagen, como al notable progreso que se ha visto en los últimos años utilizando CNN de manera más general para lograr el estado de los resultados de última generación en un conjunto de tareas de visión artificial como la detección de objetos y el reconocimiento facial.
	El modelado de datos de imagen significa que el espacio latente, la entrada al generador, proporciona una representación comprimida del conjunto de imágenes o fotografías utilizadas para entrenar el modelo. También significa que el generador genera nuevas imágenes o fotografías, proporcionando una salida que los desarrolladores o usuarios del modelo pueden ver y evaluar fácilmente.
	Puede ser este hecho por encima de otros, la capacidad de evaluar visualmente la calidad de la salida generada, lo que ha llevado al enfoque de las aplicaciones de visión por computadora con CNN y a los saltos masivos en la capacidad de las GAN en comparación con otros modelos generativos, basados en deep learning (aprendizaje profundo) o no.

GAN condicionales
	Una extensión importante de la GAN está en su uso para generar condicionalmente una salida.
	El modelo generativo se puede entrenar para generar nuevos ejemplos del dominio de entrada, donde la entrada, el vector aleatorio del espacio latente, se proporciona (condicionado) alguna entrada adicional.
	La entrada adicional podría ser un valor de clase, como masculino o femenino en la generación de fotografías de personas, o un dígito, en el caso de generar imágenes de dígitos manuscritos.


	“Las redes generativas adversarias pueden ser extendidas a un modelo condicional, si tanto el generador como el discriminador están condicionados a alguna información adicional y podrías ser cualquier tipo de información auxiliar, como etiquetas de clase o datos de otras modalidades. Podemos realizar el condicionamiento alimentando y tanto en el discriminador como en el generador como [una] capa de entrada adicional”.

El discriminador también está condicionado, lo que significa que se le proporciona una imagen de entrada que es real o falsa y la entrada adicional. En el caso de una entrada condicional de tipo etiqueta de clasificación, el discriminador entonces esperaría que la entrada fuera de esa clase, enseñando al generador a generar ejemplos de esa clase para engañar al discriminador.
De esta forma, se puede utilizar una GAN condicional para generar ejemplos a partir de un dominio de un tipo determinado.
Llevados un paso más allá, los modelos GAN pueden ser condicionados a un ejemplo del dominio como una imagen. Esto permite aplicaciones de GAN como la traducción de texto a imagen o la traducción imagen a imagen. Esto permite algunas de las aplicaciones más impresionantes de las GAN, como la transferencia de estilo, la coloración de fotos, la transformación de fotos de verano a invierno o de día a noche.
En el caso de las GAN condicionales para la traducción de imagen a imagen, como la transformación del día en noche, el discriminador recibe como entrada ejemplos de fotos nocturnas reales y generadas, así como fotos diurnas reales (condicionadas). El generador cuenta con un vector aleatorio del espacio latente, así como con fotos reales diurnas (condicionadas) como entrada.
